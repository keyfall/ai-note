#### 特征描述
- 几何描述：
	- 距离：欧式距离、街区距离、棋盘距离、明氏距离、马氏距离。
	- 位置：区域在图像中的位置用区域面积的中心点来表示。二值图像质量分布均匀，质心和形心重合。
	- 方向：将最小二阶矩轴定义为较长物体的方向
	- 尺寸、周长、面积
	- 孔数和欧拉数：孔数是指不包含感兴趣像素的被封闭边缘包围的区域；图像中的对象数减去这些对象中的孔数，即是欧拉数。
	- 凸包：点集的凸包是指一个最小凸多边形，满足点集中或在多边形边上或者在其内。
- 形状描述：
	- 矩形度：矩形度就是物体呈现矩形的程度，通常用物体对其外接矩形的充满程度来衡量。
	- 圆形度：可以采用圆度、边界能量、圆形性及内切圆、外接圆半径比等特征描述。
	- 中轴变换：中轴也称对称轴或骨架，既能压缩图像信息，又能完全保留目标的形状信息，且由中轴及其他数值还可以恢复区域，是一种重要的形状特征。
- 边界描述：
	- 链码：用于表示由顺次连接的具有指定长度和方向的直线段组成的边界线
	- 边界分段：是将边界分成若干段，分别对每一段进行表示。特点是降低边界复杂程度，简化表示过程特别适用于边界具有多个凹点的情况
	- 多边形近似：多边形的边可用线性关系表示。对一条闭合曲线，当多边形的边数等于边界上的点数时，这种近似是准确的，此时，每对相邻点定义多边形的一条边。多边形近似问题常用聚合技术和拆分技术来实现
	- 标记图是一种用一维函数表达二维边界的方法，以达到降低表达难度的目的。它可以用多种方法生成。最简单的方法是把从重心到边界的距离作为角度的函数来标记。该方法不受边界平移的影响,但是当边界旋转或发生尺度变换时，标记将会发生改变。可以用归一化方法进行解决
- 矩描述：
	- 图像矩：描述图像形状特征的一种数学工具。它们可以提供关于图像的多种信息，如大小、方向、重心等![[Pasted image 20250418162658.png]]
	- 中心矩:是对原始矩的一种改进，它通过将坐标系原点移到图像的质心（或称重心）位置来计算矩，从而使得矩对平移具有不变性。中心矩具有平移不变性，归一化后的中心矩具有比例变换不变性。![[Pasted image 20250418162707.png]]
- 纹理：
	- 灰度共生矩阵法是对图像所有像素进行统计调查，以便描述其灰度分布的一种方法
		- 构造灰度共生矩阵：统计在特定方向（如0°, 45°, 90°, 135°）上，两个像素灰度值 𝑖i 和 𝑗j 同时出现的频率。
		- 提取特征：从GLCM中提取纹理特征，如对比度（Contrast）、相关性（Correlation）、能量（Energy）、熵（Entropy）等。
	- 灰度差分统计法是考查相距微小距离的两点的灰度，计算灰度差考查概率
	- 行程长度统计法是指在同一方向上具有相同灰度值或灰度差别在某个范围内的像素个数，通过统计行程长度来体现纹理特性
	- LBP算子是一种用来描述图像局部纹理特征的算子，具有旋转不变性和灰度不变性等显著的优点，通过比较中心像素与其邻域像素的灰度值，当前中心像素生成8位二进制码(通过与领域像素比较，领域像素大于中心像素标记1，否则标记0)，再转换成十进制，作为像素的LBP
	```
	import numpy as np
	import cv2
	from skimage.feature import greycomatrix, greycoprops, local_binary_pattern
	from skimage.measure import shannon_entropy
	
	# 读取图像并转为灰度图
	image = cv2.imread("texture_image.jpg", cv2.IMREAD_GRAYSCALE)
	
	# 1. 灰度共生矩阵法 (GLCM)
	def glcm_features(image):
		distances = [1]
		angles = [0, np.pi/4, np.pi/2, 3*np.pi/4]
		glcm = greycomatrix(image, distances=distances, angles=angles, levels=256, symmetric=True, normed=True)
		contrast = greycoprops(glcm, 'contrast')
		correlation = greycoprops(glcm, 'correlation')
		energy = greycoprops(glcm, 'energy')
		homogeneity = greycoprops(glcm, 'homogeneity')
		return {
			"Contrast": contrast,
			"Correlation": correlation,
			"Energy": energy,
			"Homogeneity": homogeneity
		}
	
	# 2. 灰度差分统计法
	def gray_diff_stats(image):
		diff = np.abs(np.diff(image.astype(float), axis=1))
		mean_diff = np.mean(diff)
		var_diff = np.var(diff)
		return {
			"Mean Difference": mean_diff,
			"Variance Difference": var_diff
		}
	
	# 3. 行程长度统计法
	def run_length_stats(image):
		runs = []
		for row in image:
			current_run = 1
			for i in range(1, len(row)):
				if row[i] == row[i - 1]:
					current_run += 1
				else:
					runs.append(current_run)
					current_run = 1
			runs.append(current_run)
		short_run_emphasis = np.sum([1 / (r**2) for r in runs]) / len(runs)
		long_run_emphasis = np.sum([r**2 for r in runs]) / len(runs)
		return {
			"Short Run Emphasis": short_run_emphasis,
			"Long Run Emphasis": long_run_emphasis
		}
	
	# 4. LBP算子
	def lbp_features(image):
		radius = 3
		n_points = 8 * radius
		lbp = local_binary_pattern(image, n_points, radius, method="uniform")
		hist, _ = np.histogram(lbp.ravel(), bins=np.arange(0, n_points + 3), range=(0, n_points + 2))
		hist = hist.astype("float")
		hist /= (hist.sum() + 1e-6)  # 归一化
		return hist
	
	# 输出结果
	if __name__ == "__main__":
		print("GLCM Features:")
		print(glcm_features(image))
	
		print("\nGray Difference Statistics:")
		print(gray_diff_stats(image))
	
		print("\nRun Length Statistics:")
		print(run_length_stats(image))
	
		print("\nLBP Histogram:")
		print(lbp_features(image))
	```
#### 特征提取
- SIFT算法(cv2.SIFT_create())
	- 原理
		- 尺度空间极值检测：首先通过高斯模糊对输入图像进行不同尺度的平滑处理，构建图像的尺度空间(通过不同高斯核与图片卷积，进行不同程度的平滑处理的程度)。把尺度空间的图片作为高斯金字塔每一层，每一层对于图片进行降采样(一般3-5次)，相当于每一层是一个小的图像金字塔，这里叫小层，不同尺度的层叫大层，寻找候选关键点，每个采样点和当前采样点四周8个点加上上下大层间当前位置的9个点，总共8+2×9=26个点进行比较，找到最大点，就认为是一个候选关键点
		- 关键点选取定位：找到候选关键点后，需要进一步精确定位它们的位置并去除低对比度的关键点和边缘响应过强的关键点。这一步骤包括使用泰勒展开式精确计算极值点的位置，以及应用Hessian矩阵判断关键点是否位于边缘。
		- 方向赋值:为每个确定的关键点分配一个或多个主方向，基于关键点邻域像素的梯度方向分布来实现。计算以关键点为中心的一个邻域内所有像素的梯度大小和方向，然后创建一个以梯度方向为变量的直方图，直方图的峰值对应的梯度方向即为该关键点的方向。确定关键点周边的主方向![[Pasted image 20250422191754.png]]
		- 关键点描述符生成：将关键点周围区域分成4×4子区域，然后在每个子区域内统计8个方向，形成一个向量，总共4×4×8=128维，这个向量即为关键点的描述符，这里是局部的细节信息，提高匹配的准确性和鲁棒性，
		  它可以用来匹配不同图像中的关键点。![[Pasted image 20250422191809.png]]
	- 优点：对尺度、旋转、亮度变化具有不变性；对视角变化和遮挡鲁棒,拥有抗噪能力，区分度
	
		|**特性**|**实现步骤**|
		|---|---|
		|**尺度不变性**|高斯金字塔构建 + DoG 关键点检测 + 子像素精确定位|
		|**旋转不变性**|梯度方向直方图 + 主方向分配 + 描述符旋转对齐|
		|**光照不变性**|局部对比度归一化 + 梯度幅值截断|
	- 缺点：计算复杂度高；专利保护（有限制）
	- 使用场景：图像匹配、3D模型建立
- SURF:
	- 原理：
		- 兴趣点检测：通过Hessian矩阵，积分图，盒子滤波器确定图像中每个像素点是否为关键点。SURF利用盒子滤波器(Box Filter)近似高斯二阶导数滤波器来加速计算过程。![[Pasted image 20250424223023.png]]
		- 尺度空间构建：使用不同尺度的盒子滤波器对图像进行卷积操作，以识别出在不同尺度下稳定的特征点。
		- 关键点定位：非极大值抑制技术被用来精确定位关键点的位置，并去除低对比度的关键点和边缘响应过强的关键点。![[Pasted image 20250424235409.png]]
		- 方向赋值：对于每一个关键点，计算其周围像素的Haar小波响应，然后根据这些响应的方向分布为主关键点分配一个或多个主方向。
		- 特征描述子生成：围绕每个关键点，在选定的尺度上，考虑其主方向，提取一个64维（简化版本）或128维的特征向量作为该点的描述子。
	- 优点：
		- 速度更快：由于采用了积分图技术和盒子滤波器，使得SURF在计算效率方面显著优于SIFT。
		- 稳健性好：SURF保持了尺度和旋转不变性，并且对光照变化具有较好的鲁棒性。
		- 实时处理能力：适合应用于需要实时处理的场合，如视频流中的物体跟踪。
	- 缺点: 精度略逊于SIFT,虽然速度更快，但在复杂场景中(噪声大，复杂纹理，尺度变化大，旋转角度大，光照变化强烈，图像变换大，特征点数量少)，其特征匹配的精确度不如SIFT。
	- 使用场景：
		- 图像检索：在大规模图像数据库中查找相似的图像。
		- 目标识别与跟踪：实时监控系统中的人脸或物体识别与跟踪。
		- 全景拼接：将多张照片自动拼接成一张全景图。
		- 增强现实(AR)：将虚拟信息叠加到真实世界视图中的应用。
	- SIFT区别：

	|特性/方面|SIFT (Scale-Invariant Feature Transform)|SURF (Speeded-Up Robust Features)|
	|---|---|---|
	|**特征点检测**|使用Difference of Gaussians (DoG)在尺度空间中寻找极值点|利用Hessian矩阵（基于盒子滤波器和积分图）快速检测关键点|
	|**尺度空间表示**|通过改变高斯模糊的程度构建尺度空间|使用不同大小的盒子滤波器直接作用于积分图像上，加速多尺度分析|
	|**方向赋值**|根据梯度方向直方图分配主方向|计算关键点周围Haar小波响应的方向分布来确定主方向|
	|**特征描述子生成**|128维特征向量，基于关键点邻域内的梯度方向和大小|64维或128维特征向量，基于关键点邻域内的Haar小波响应|
	|**计算效率**|相对较慢，尤其是在处理大规模数据集时|显著加快，适用于实时应用|
	|**专利状态**|开源|曾有专利限制，但现已开源|
	|**鲁棒性和独特性**|对光照、旋转等变化具有良好的适应性|在保持相似性能的同时提高了速度，并在某些情况下增强了稳健性|
- BRISK:
	- 原理：
		 -  特征点检测：BRISK使用AGAST（Adaptive and Generic Automated Speeded up Test）算法进行特征点检测。AGAST是FAST角点检测器的一个变种，优化了速度和性能。(FAST:图像中的每一个像素点p，如果它周围的连续n个像素（通常n=9或12）中，有超过t个像素的亮度要么都比p亮很多（大于阈值），要么都比p暗很多（小于阈值），则认为p是一个角点。AGAST就是会自适应n和t的值)
		 - 尺度空间表示：为了确保尺度不变性，BRISK在不同尺度上(通过高斯金字塔)检测特征点，并为每个特征点计算一个尺度值(尺度值是高斯核标准差σ)。
		 - 描述符构建：对于每个检测到的特征点，BRISK通过比较该点周围采样点对之间的强度来构造一个二进制描述符。这些采样点对按照特定模式(环状，螺旋状)分布在特征点周围，并且这种模式考虑了尺度信息。通过对这些点对进行相对亮度比较（比值大于或小于某个阈值或者灰度值比较），形成一系列比特位，最终构成一个512位的二进制字符串作为描述符。
		 - 匹配：使用汉明距离来度量两个二进制描述符之间的相似度，从而实现特征点的匹配。
	- 优点：
		- **速度快**：由于采用了二进制描述符和高效的特征点检测方法，BRISK算法在处理速度上有明显优势。
		- **内存占用低**：二进制描述符相较于浮点数描述符占用更少的存储空间。
		- **旋转不变性和尺度不变性**：设计时充分考虑了旋转和尺度变化的影响，使得算法具有良好的鲁棒性，利用关键点周围的采样点之间的相对位置和亮度差异来计算一个主导方向(通过亮度差和加权梯度得到采样点对的矢量，全部相加后得到总方向矢量，主方向角度=arctan(Gy/Gx),)。这个主导方向用于将描述符调整到一致的方向，从而实现旋转不变性。![[Pasted image 20250425010618.png]]
	- 缺点：
		- 复杂场景区分能力差
		- 对噪声敏感
	- 使用场景
		- **物体识别与跟踪**：通过在连续帧或不同视角下的图像之间找到相同的特征点，可以追踪物体的位置和姿态变化。
		- **场景重建**：利用多视角下的特征点匹配结果，可以从二维图像中恢复三维结构。
		- **图像配准与拼接**：基于特征点的匹配，可以将多幅图像对齐或拼接在一起，创建全景图或其他复合图像。
		- **相机标定**：特征点匹配还可以帮助确定相机的内部参数和外部参数，这对于许多高级应用至关重要。
- FREAK：
	- 原理：特征点描述算法
		- **关键点检测**：首先，FREAK需要一个关键点检测器来识别图像中的显著点。虽然FREAK本身不提供关键点检测功能，但它通常与FAST或BRISK等检测器一起使用。
		- **采样模式应用**：一旦确定了关键点的位置，就会在其周围应用一组预先定义好的采样模式。每个模式由一对或多对像素组成，这些像素对分布在以关键点为中心的不同半径上。FREAK使用了43个这样的采样模式，从中心向外扩展。![[Pasted image 20250425004910.png]]
		- **二进制编码**：和BRISK一样，对于采样点进行强度比较，生成512位二进制编码
	- 优点：
		- 1**高效性**：FREAK使用二进制描述符，这使得特征匹配过程非常快，适合实时应用。
		- **鲁棒性**：相对于其他一些特征描述方法，FREAK在面对尺度变化、旋转、光照变化等情况下表现出较好的鲁棒性。
		- **低内存占用**：二进制描述符比浮点型描述符占用更少的存储空间，有利于在资源受限的环境中使用。
	- 缺点：
		- **性能限制**：虽然FREAK在很多场景下都表现良好，但在某些极端条件下（如非常大的尺度变化或严重的视角变换），其性能可能不如SIFT或SURF等传统算法。
		- **依赖底层特征检测器**：FREAK本身是一个特征描述符，通常需要与其他关键点检测算法（如FAST、BRISK）结合使用，这意味着它的性能也受到所选关键点检测算法的影响。
	- 使用场景：
		-  视频流中的物体跟踪
		- 增强现实应用，其中需要快速准确地识别和跟踪现实世界中的对象
		- 移动机器人导航与定位，特别是当计算资源有限时
		- 图像检索系统，尤其是需要处理大规模数据集的情况
- **MSER**：
	- 原理：特征检测算法，主要用于提取图像中的稳定区域
		- **灰度阈值化**：对输入的灰度图像应用一系列从低到高的阈值，形成一系列二值图像。
		- **连通域检测**：在每个阈值下，识别所有连通区域。
		- **稳定性评估**：对于每个连通区域，在不同的阈值下跟踪其大小变化，并计算其面积随阈值增加的变化率。如果某个区域在其大小变化率低于某个阈值时被认为是“最大稳定”的，则该区域被标记为MSER。
		- **过滤和后处理**：去除过小或过大的区域，并进行必要的几何变换等后处理步骤。![[Pasted image 20250425022509.png]]
	- 优点：
		- **仿射不变性**：MSER具有良好的仿射变换不变性，这意味着它能够有效应对视角变化带来的影响。
		- **光照不变性**：由于其依赖于相对的像素强度而不是绝对值，因此对光照变化有较好的鲁棒性。
		- **高效性**：相比其他特征检测方法，MSER的计算效率较高。
	- 缺点：
		-  **尺度敏感**：虽然MSER对仿射变换具有不变性，但它对尺度变化较为敏感。
		- **噪声敏感**：在高噪声环境下，MSER的表现可能会受到影响。
		- **实现复杂度**：与一些简单的特征检测器相比，MSER的实现可能更复杂。
	- 使用场景：
		- - **文字检测**：MSER常用于自然场景中的文字检测，因为它能很好地适应不同的字体、大小和背景条件。
		- **物体识别**：适用于需要检测和识别具有显著纹理或颜色对比度的物体。
		- **医学图像分析**：可用于识别X光片、CT扫描等医学图像中的特定结构。
		- **视频分析**：可用于追踪视频中出现的感兴趣区域，尤其是在监控系统中。
- HOG：
	- 原理：特征描述符
		- **灰度化**：将输入的彩色图像转化为灰度图像。
		- **Gamma校正**：对图像进行Gamma校正，目的是减少光照变化的影响。
		- **计算梯度**：利用一阶微分算子（如Sobel算子）计算每个像素点在x方向和y方向上的梯度值。
		- **构建梯度方向直方图**：将图像划分为若干个单元格（cell），对于每个cell中的每个像素，根据其梯度方向将其累加到对应的bin上，形成该cell的方向梯度直方图。
		- **归一化处理**：为了增强对比度不变性，通常会对相邻的几个cell组成的块（block）进行归一化处理。
		- **特征向量生成**：所有块的梯度直方图连接起来形成最终的特征向量。
	- 优点：
		- 对于形状和外观的变化具有较强的鲁棒性。
		- 在不同尺度下表现良好，尤其适合行人检测等应用。
		- 计算效率较高，可以实时处理。
	- 缺点：
		- 对于遮挡、背景杂乱等情况敏感。
		- 不擅长捕捉细微的纹理变化。
		- 参数设置（如cell大小、block大小等）需要根据具体情况调整，缺乏通用性。
	- 使用场景：
		- - **行人检测**：由于HOG能够很好地捕捉人体轮廓信息，因此在行人检测方面应用非常广泛。
		- **物体识别**：除了行人之外，还可以用于其他特定类型的物体识别任务，如车辆、动物等。
		- **安全监控**：可用于视频监控系统中的人体行为分析、异常行为检测等。
- ORB：
	- 原理：
		- **特征点检测（FAST角点检测）**：首先使用FAST算法来寻找图像中的关键点。为了增加特征点的独特性和分布均匀性，ORB在此基础上引入了一种非极大值抑制策略(通过关键点与旁边点的达到条件的响应数量作为比较规则)以及多尺度特性(图像金字塔上进行FAST算法)。
		- **特征点方向确定（Orientation by Intensity Centroid）**：为每个找到的关键点分配一个主方向，这通过计算强度质心并由此确定方向来实现。这样做是为了让ORB能够支持旋转不变性。
		- **生成描述子（Rotated BRIEF）**：对于每个关键点，根据其主方向调整采样模式，然后应用BRIEF算法，使用256个位置对(通过主成分分析得到的，原始brief随机生成)生成256位描述子，。由于BRIEF本身不具有旋转不变性，因此ORB通过对采样模式进行旋转来补偿这一点，从而使得描述子具有一定的旋转不变性。
		- **匹配**：最后，使用汉明距离作为相似度度量标准来比较不同图像之间的描述子，以找到匹配的关键点。
	- 优点：
		- **速度快**：ORB在速度方面表现出色，特别适合实时应用。
		- **计算效率高**：相较于SIFT、SURF等算法，ORB的计算资源需求更低。
	- 缺点：
		- **性能较弱**：在一些复杂的变换情况下（如较大的视角变化或光照变化），ORB的表现可能不如SIFT或SURF。
		- **描述子独特性较低**：ORB使用的BRIEF描述子相对较短，可能导致匹配时出现更多的误匹配。
	- 使用场景:需要快速处理大量数据的应用场景，特别是在硬件资源有限的情况
- HAAR：
	- 原理：
		- **积分图计算：** 积分图使得任何矩形区域内像素之和可以在常数时间内计算完成。
		- **特征提取：** HAAR特征基于相邻矩形区域像素和的差值。包括边缘特征、线性特征、中心环绕特征等。
		- **Adaboost选择特征：** 使用Adaboost算法从大量候选特征中挑选出最能区分目标与非目标的特征，并将其组合成强分类器。
		- **级联分类器：** 将多个强分类器串联起来形成一个级联结构，每个阶段都快速排除背景区域，从而提高处理速度。
	- 优点：
		- **速度快：** 通过使用积分图和Adaboost算法，能够在实时应用中高效地进行对象检测。
		- **简单有效：** 特征简单，易于计算，但对许多对象类型非常有效，特别是人脸检测。
		- **可扩展性强：** 可以训练检测不同类型对象的分类器。
	- 缺点：
		- **准确性有限：** 对于复杂背景或需要高精度定位的任务，可能不如深度学习方法准确。
		- **对光线变化敏感：** HAAR特征在光线条件变化较大时性能可能会下降。
		- **依赖特征设计：** 效果很大程度上取决于手工设计的特征是否适合特定的应用场景。
	- 使用场景：
		- **人脸检测：** HAAR级联是最早被广泛应用于人脸检测的技术之一，至今仍在很多场合中使用。
		- **初步筛选：** 在一些需要快速过滤掉大部分非目标区域的应用中，HAAR可以作为第一道筛选工具。
		- **实时系统：** 需要实时处理的应用，如视频监控系统中的人脸检测，HAAR算法因其高效性而被采用。
- 区别

|算法|基本原理|优势|缺点|
|---|---|---|---|
|SIFT|尺度空间中寻找极值点，并使用直方图统计梯度方向建立描述符。|对于尺度变化、旋转具有不变性；对光照变化较稳定。|计算复杂度高；专利限制（虽然在某些国家现已过期）。|
|SURF|使用积分图像加速Hessian矩阵的计算，并用快速傅里叶变换来生成描述符。|比SIFT更快；对尺度和旋转具有不变性。|专利问题；对于大的视角变化敏感。|
|BRISK|通过比较一对位置上的像素强度构建二进制字符串作为描述符。|快速且高效；对噪声有一定鲁棒性。|描述符长度较长时性能下降；对较大的尺度变化敏感。|
|FREAK|受人类视觉系统启发，采用一组定向模式来构造二进制描述符。|高效；具有良好的匹配性能；对噪声有较好的鲁棒性。|对较大的尺度和视角变化较为敏感。|
|MSER|寻找图像中的稳定区域，即最大稳定极值区域，作为特征点。|对于仿射变换具有不变性；适合用于纹理较少的区域。|对于高斯噪声较为敏感；提取的特征数量可能不稳定。|
|HOG|计算图像局部区域的梯度方向直方图，用以描述物体形状。|对于光照变化和阴影具有较强的鲁棒性；适用于行人检测等应用。|对于复杂的背景和遮挡处理能力有限；不考虑颜色信息。|
|ORB|结合FAST关键点检测器和旋转BRIEF描述符，并加入了多尺度特性。|免费开源；运算速度快；对噪声有较好的鲁棒性。|对于较大的视角变化敏感；性能略低于SIFT和SURF。|
|HAAR|使用矩形特征表示图像的不同部分，通常结合AdaBoost等机器学习方法进行分类。|计算简单；可用于实时应用；广泛应用于人脸检测等领域。|特征表达能力有限；对光线变化和角度变化较为敏感。|
